# 05 — Print Formats

All print formats are version-controlled (`standard=1`) under
`edupro_sms/edupro_sms/print_format/<name>/`.

## 5.1 IGCSE Report Card

Print format on the `Report Card` DocType (`docs/03_DocTypes.md`).
Rendered to PDF via Frappe's native `frappe.get_print` +
`frappe.utils.pdf.get_pdf` pipeline (wkhtmltopdf under the hood).

Layout, top to bottom:

1. **Header** — school name + motto from `School Settings`, "TERM REPORT
   CARD — {term}, {year}".
2. **Student details block** — student name, class, term, academic year.
3. **Academic performance table** — one row per subject (from the
   `assessment_results` child table): Subject, Score (total/max), Grade,
   Comment. Footer row: total, average %, overall grade, class position.
4. **Class Teacher comment** block.
5. **Headmaster comment** block.
6. **Authenticity QR code** — generated by `edupro_sms/qr.py`, links to
   `/verify-report-card?code=<verification_code>` so a printed copy can
   be checked against the live record.
7. **Signature blocks** — Class Teacher / Headmaster signature lines.
8. **Footer** — generation date, school phone/email from `School
   Settings`, watermark (`edupro_sms/watermark.py`).

## 5.2 Fee Statement

Print format on the `Student Fee` DocType. Renders as a ledger statement
showing the running Debit/Credit/Balance for a student, sourced from
`Student Ledger Entry` records.

Layout, top to bottom:

1. **Header** — school name + motto from `School Settings`, "FEE
   STATEMENT".
2. **Student details block** — student name, admission number, class.
3. **Ledger table** — one row per `Student Ledger Entry` (ordered
   `posting_datetime` ascending): Date, Description, Debit, Credit,
   Balance.
4. **Summary block** — total owed, total paid, balance due.
5. **Footer** — bursar contact details from `School Settings`.

## 5.3 Payment Receipt

Print format for a single payment event, generated when the Bursar
records a payment against a `Student Fee`. Shows the amount received,
the resulting balance, and student/school identifying details.

## 5.4 Marks Entry Screen (reference, not a print format)

The primary teacher data-entry surface — `www/marks-entry/`, not a
Print Format, but documented here since it's the counterpart to what
ends up on the printed report:

- Header: class, term, subject, current status, Save Draft / Submit for
  Approval actions.
- Table: one row per student — Term Mark, Exam Mark, auto-calculated
  Total, auto-calculated Grade, subject comment field.
- Inline validation flags students with missing marks before allowing
  Submit for Approval.

## 5.5 PDF Generation Notes

- Uses Frappe's native Print Format → PDF pipeline, not a separate PDF
  library — keeps rendering inside the framework's print system.
- wkhtmltopdf needs a container-reachable base URL to fetch print CSS —
  in Frappe Cloud this is handled by the platform; in local Docker dev,
  set `host_name` to a container-reachable address if PDFs render
  without styling.
